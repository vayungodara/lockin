## Notes

- The shared `createMockSupabase()` builder returns the same mock value for all chained queries. This limits testing of functions that make multiple sequential Supabase calls (e.g., `getGroupActivity` fetches activity, then profiles, then reactions). For better coverage of these multi-call functions, consider adding a sequential mock helper or per-table mocking.
- `partnerships.js` functions heavily use `.or()` for querying both sides of a partnership. The mock was extended to support this.
- `onboarding.js` `detectProgress` uses `Promise.all` with 4 parallel Supabase queries, all returning the same mock value — limits the realism of the test but validates the function structure and error handling.
- `unlockAchievement` and `checkPactAchievements` in `gamification.js` were not tested because they use dynamic `import('@/lib/notifications')` and make multiple sequential Supabase calls that are hard to mock distinctly with the current shared builder pattern. Consider adding per-call mock sequencing for future coverage.
